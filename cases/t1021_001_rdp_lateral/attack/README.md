# Attack reproduction — RDP pivoting

After credential acquisition (Mimikatz, LSASS dump, etc.), use mstsc or
FreeRDP from the compromised host to RDP into multiple internal Windows
machines in sequence. Zeek records the conn.log entries; the detection
in `../detection/` aggregates and fires.

## Prereqs

- Compromised lab host with cached credentials
- Multiple internal Windows machines reachable on TCP/3389
- Lab compose stack running

## Run (high level)

```powershell
# Windows mstsc — interactive, one host at a time
mstsc /v:192.168.1.10
# (auth, work, disconnect, repeat for next host)

# Or scripted via FreeRDP
xfreerdp /u:admin /p:Password123 /v:192.168.1.10
xfreerdp /u:admin /p:Password123 /v:192.168.1.20
xfreerdp /u:admin /p:Password123 /v:192.168.1.30
```

## Capture telemetry into a fixture

```bash
docker exec detlab-zeek cat /usr/local/zeek/logs/current/conn.log \
  | jq -c "select(.\"id.resp_p\" == 3389 and .\"id.orig_h\" == \"<attacker_ip>\")" \
  > /tmp/rdp_lateral_capture.log
head -50 /tmp/rdp_lateral_capture.log > tests/positive_conn.log
```

## Safety / scope

- Lab use only.
- The synthetic `tests/positive_conn.log` shipped here is generated by
  `scripts/generate_fixtures.py` so the test suite is hermetic.
