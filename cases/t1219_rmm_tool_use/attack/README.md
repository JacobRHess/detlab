# Attack reproduction — RMM tool installation

Install AnyDesk (or any other RMM tool) on a lab workstation, accept the
default cloud-connect, watch DNS resolve the vendor domains, fire the
detection.

## Prereqs

- Windows or macOS lab host on the lab subnet
- Lab compose stack running so Zeek is capturing

## Run (high level)

```powershell
# Windows — silent install
choco install anydesk -y

# Launch and accept the click-to-allow remote-control prompt
"C:\Program Files (x86)\AnyDesk\AnyDesk.exe"
```

```bash
# macOS
brew install --cask anydesk
open -a AnyDesk
```

The client immediately resolves `boot.anydesk.com`, `relay-fra.anydesk.com`,
and a region-specific relay; that traffic is what fires the detection.

## Capture telemetry into a fixture

```bash
docker exec detlab-zeek cat /usr/local/zeek/logs/current/dns.log \
  | jq -c "select(.query | test(\"anydesk|teamviewer|screenconnect\"))" \
  > /tmp/rmm_capture.log
head -50 /tmp/rmm_capture.log > tests/positive_dns.log
```

## Refreshing the lookup

```bash
# Curate from a maintained list — Red Canary publishes one quarterly
curl -fsSL https://github.com/redcanaryco/atomic-red-team/raw/master/atomics/Indexes/Indexes-CSV/index.csv \
  | grep RMM \
  | awk -F, '{print $5}' \
  > /tmp/rmm_domains.txt
```

## Safety / scope

- Lab use only.
- All listed RMM tools are legitimate commercial products; running them
  on a host you own is not malicious. The detection content treats
  resolution of these domains as **suspicious in context**, not malicious
  per se — that's what the allowlist is for.
- The synthetic `tests/positive_dns.log` shipped here is generated by
  `scripts/generate_fixtures.py` so the test suite is hermetic.
