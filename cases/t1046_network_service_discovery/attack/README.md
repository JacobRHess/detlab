# Attack reproduction — nmap port scan

Run nmap from a host on the lab subnet against another lab host. Zeek will
record one `conn.log` entry per probed port; that telemetry fires the
detection in `../detection/`.

## Prereqs

- Attacker host on the lab bridge network (any Linux/Windows with nmap)
- Target host on the same bridge — even an empty Linux container is fine
- Lab compose stack running so Zeek + Splunk are capturing

## Run (high level)

```bash
# Default SYN scan, top-1000 ports — fires the detection cleanly
nmap -sS 192.168.1.10

# Wider sweep that fires faster
nmap -sS -p1-65535 --min-rate=500 192.168.1.10

# Slow / stealthy scan that ducks the default thresholds
nmap -sS -T2 --max-rate 50 -p1-1024 192.168.1.10
```

## Capture telemetry into a fixture

```bash
# Right after the scan finishes:
docker exec detlab-zeek cat /usr/local/zeek/logs/current/conn.log \
  | jq -c "select(.\"id.orig_h\" == \"<attacker_ip>\")" \
  > /tmp/portscan_capture.log

head -300 /tmp/portscan_capture.log > tests/positive_conn.log
```

## Safety / scope

- Lab use only.
- Port scanning a host you don't own may be illegal in your jurisdiction.
- The synthetic `tests/positive_conn.log` shipped here is generated by
  `scripts/generate_fixtures.py` so the test suite is hermetic — replace
  with a real capture once you have one.
